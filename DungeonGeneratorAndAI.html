Dungeon Generator And AI

This assignment was about generating a dungeon and making an AI agent that can move trough the dungeon. I made 2 AI agent with different behaviors. The dungeon generator needed a few inputs from the user like the size of the dungeon and how many rooms the dungeon needed to generate. After that it automatically generated a dungeon. By changing “the seed” the user could make different dungeons with the same inputs.

My dungeon generator is based on characters. That means that each piece of my dungeon is linked to a certain character. So characters like ─ │ ┌ ┐ stands for a horizontal dungeon piece a vertical, a corner to right and a corner piece to the left. Then I had to write out for each character which characters he could have as neighbors. After that, I made a 2d array that represents the dungeon (see images). First I fill the array with the amount of rooms I want to have. The next step is to fill in the rest of the array. I iterate over each array slot, that is not filled yet, and try to find the best character that fits in that specific array slot according to the neighbor rules we made earlier. Finally I check if there are any dead ends. If so, I close de dungeon with a dead end prefab and I check if there are unreachable parts in the dungeon and remove those parts. 


public void InitializeDungeon(){
    dungeon = new char[mapX, mapY];
    InitializeWalls();

    for (int y = 1; y < mapY - 1; y++){
        for (int x = 1; x < mapX - 1; x++){
            dungeon[x, y] = 'O';
        }
    }

    Random.InitState(seed);
    CreateRooms();

    for (int y = 1; y < mapY - 1; y++){
        for (int x = 1; x < mapX - 1; x++){
            if (dungeon[x, y] == '@' || dungeon[x, y] == '˄' || dungeon[x, y] == '˂' || dungeon[x, y] == '˃' || dungeon[x, y] == '˅' || 
                dungeon[x, y] == '╔' || dungeon[x, y] == '╧' || dungeon[x, y] == '╤' || dungeon[x, y] == '╗' || dungeon[x, y] == '╟' || 
                dungeon[x, y] == '╢' || dungeon[x, y] == '╚' || dungeon[x, y] == '╝')
            { //These are the room characters
                continue;
            }

            string valid = GetValidCharacters(x, y);
            dungeon[x, y] = valid[Random.Range(0, valid.Length)];
        }
    }

    FillDeadEnds();
}

Regarding the AI ​​I implemented a GOAP (Goal Oriented Action Planning) agent. How does the algorithm know which actions go together to satisfy a goal? Actions and goals use conditions, which consists of a string and a value. The action 'Attack Ranged' for example has the precondition [ "hasRangedWeapon", true ]. The action 'Equip Ranged Weapon' has the effect [ "hasRangedWeapon", true ]. Thus the algorithm knows to first equip the ranged weapon, and then use it to attack. This leads to natural, dynamic behavior without ever having to code any transitions! 


public void ExecuteGoal(){
    if (currentGoal == null){
        ReplanGoal();
    }
    else if (currentAction == null){
        GetNextAction();
    } 
    else if (currentAction.IsViable() && currentGoal.IsViable(state)){
        currentAction.OnUpdateAction();
    }
    else{
        //Current Goal or Action no longer viable! Replan!
        ReplanGoal();
    }
